Доступ к API

Авторизация и аутентификация осуществляются индивидуальным клиентским токеном, который выдается Юриком при подключении. У каждого токена есть время начала и завершения действия.

Токен передается как часть URL, поэтому держите настройки и логи в тайне.

Обращения к API

Все обращения к API должны быть в виде POST запросов с телом в формате JSON. Ответы от API так же передаются формате JSON. URL запроса имеет вид https://api.yoorik.work/{version}/{token}/{action}, где:

  • version - версия API, которую вы используете (пока только tricepsV1);
  • token - токен, выданный Юриком;
  • action - одно из двух действий pushEvents или fetchProposals, которое нужно выполнить.

Ошибки

Ошибка 400

Ошибка 400 означает, что формат переданных данных некорректен и Юрику непонятен.

В ответе API в поле description указана подробная информация, включая путь валидации объектов данных, которая поможет найти проблему. После её устранения, вы можете повторить отправку данных в исправленном формате.

ВАЖНО! Не нужно предпринимать повторные попытки отправки некорректных данных. При множественных запросах с ошибками в API, Юрик может заблокировать токен.

Ошибка 403

Ошибка 403 означает, что токен не валиден. Возможно допущена опечатка или истекло время его действия.

Ошибки 50х

Ошибки 500 и выше, означают, что Юрик переутомился, и ему нужно дать отдохнуть. Подождите немного времени и повторите запрос.

У него установлены поля ok равное false и description с описанием ошибки. В случае, если установлен заголовок Accept: text/html (что типично при запросе из браузера), то ошибка возвращается в виде HTML страницы.

Получить предложения

Метод отправляет предложения по обновлению данных, если они есть.

Если передан параметр timeout и предложений нет, то метод будет ждать не более указанного количества секунд. Если предложений за это время не появится, то метод вернет пустой массив.

Рекомендуется всегда указывать в параметр after идентификатор последнего полученного предложения чтобы не пропустить новые, в случае сетевых ошибок. Если параметр не задан, будут переданы только те предложения, которые Юрик еще не присылал.
